Guidance of Simplechain Deployment and RPC Settings

How to deploy a Simplechain node

This guidance includes how to deploy, run and make RPC calls.

Suppose the readers are able to use Linux,Mac or Windows command line. If you are not familiar with command line, or just interested in using GUI wallet, please refer to another document about Simplechain Wallet Tutorial.

Get Simplechain Node

There are at least 4 ways to get a Simplechain node, you can choose any way：

1. Download the binaries

If you do not care much about Simplechain's source code, the easiest way to abtain Simplechain node is to download the latest binaries on the Simplechain release page. Currently it supports Linux, Window, OSX. It is highly recommanded to download the latest version. In this guidance, we will use v0.14.13 version as an example。

（Note: the version number you see might be different, here is v0.14.13, but other key words except for version are the same.）

  • For Mac user：Simplechain-0.14.13-osx64.tar.gz
  • For Linux users： Simplechain-0.14.13-i686-pc-linux-gnu.tar.gz(32bit) or Simplechain-0.14.13-x86_64-linux-gnu.tar.gz(64bit)
  • For Windows users：Simplechain-0.14.13-win32.zip(32bit) or Simplechain-0.14.13-win64.zip（64bit）
  • For Raspberry Pi：Simplechain-0.14.13-arm-linux-gnueabihf.tar.gz

Decompress after downloading, then you will get Simplechaind and Simplechain-cli under the path <your path>/bin.

2. apt install for Linux users

Please refer to https://github.com/Simplechainproject/Simplechain/wiki/Setting-up-Simplechain-Ubuntu,-Debian-and-Mint-repository，Current support Ubuntu，Debian and Mint。

Raspberry Pi users can also use apt，please refer to https://github.com/Simplechainproject/Simplechain/wiki/Installing-Simplechain-on-Raspberry-Pi。

After installation, you should be able to run Simplechaind and Simplechain-clidirectly in the terminal。

3. Compiler from source code

If you want to compile latest Simplechain from source code, please pull the latest source from github : https://github.com/Simplechainproject/Simplechain。

The instruction for compilation : https://github.com/Simplechainproject/Simplechain/blob/master/README.md。Currently we recommand to compile on Linux or OSX, while other platforms might need more configuration.

After compilation, you also get Simplechaind and Simplechain-cli under the path <your path>/src/

4. Get Simplechain node docker image

Suppose user has docker environment installed correctly. (what is docker?)

The Simplechain official docker image on docker hub is Simplechain/Simplechain, you can pull it by docker command ：

docker pull Simplechain/Simplechain:latest

All the four ways above cand get Simplechain nodes. There are two important binaries which are related to deployment and RPC calls:

  • Simplechaind：Simplechain core program, i.e. Simplechain fullnode program.
  • Simplechain-cli：Simplechain command line interface，interact with Simplechaind, achieve RPC calls.

Deploy Simplechain node

For the instruction for Simplechain docker images, please refer to "How to launch Simplechain with docker".

Other normal deployment methods are almost the same, here we take Ubuntu as an example. The commands in Mac and windows are exactly the same.

Run Simplechain fullnode with ./Simplechaind：

./Simplechaind -daemon

This with launch a Simplechaind daemon with the option -daemon. If user are interested in the event logs about smart contract, please add one more option -logevents.

More options can be found：

./Simplechaind -help

To stop running：

./Simplechain-cli stop

The default data path for different platforms:

  • Linux：~/.Simplechain/
  • Mac OSX：~/Library/Application Support/Simplechain
  • Windows:%APPDATA%\Simplechain

You can also use -datadir to set your own data path。

Simplechain node will sync all the historical block data for first launch, and saves to datadir. It might take some time. Then diagnostics for the node can be found under the data path ~/.Simplechain/debug.log。

Local RPC call

When the Simplechain node is running, we can use Simplechain command line interface Simplechain-cli to interact with Simplechaind, and make local RPC calls.

e.g.：

oldclock@raven:~/Simplechain-0.14.3/bin$ ./Simplechain-cli getinfo

  "version": 140300,
  "protocolversion": 70016,
  "walletversion": 130000,

    "proof-of-work": 1.52587890625e-05,

To get all RPC command list:

./Simplechain-cli help

To get help for specific RPC command, use ./Simplechain-cli help <RPCcmd>，e.g.：

./Simplechain-cli help getinfo

JsonRPC settings

You can get the jsonrpc example by using (here we use getinfo as an example): ./Simplechain-cli help getinfo

Examples:
> curl --user myusername --data-binary '{"jsonrpc": "1.0", "id":"curltest", "method": "getinfo", "params": [] }' -H 'content-type: text/plain;' http://127.0.0.1:3889/

The example shows detailed format for jsonrpc content. However, it can only be called after setting rpcuser and rpcpassword. There are two ways to set the parameters, you can choose either.

Method 1：Create config file ~/.Simplechain/Simplechain.conf，which should include:

rpcuser=test  #rpc user name, (necessary)
rpcpassword=test1234  #rpc password, (necessary)
# By default, only local jsonrpc is allowed
# Remote connection is allowed only when setting rpcallowip as following:
# both ipv4 and ipv6 can be set, e.g.:
#rpcallowip=192.168.77.51/255.255.255.0
#rpcallowip=1.2.3.4/24
#rpcallowip=2001:db8:85a3:0:0:8a2e:370:7334/96

For more config settings, please refer to ：Simplechain.conf example

After creating the config file, please restart the node to finish settings.

Method 2：Restart Simplechain node with following options:

./Simplechaind -daemon -rpcuser=test -rpcpassword=test1234 -rpcallowip=192.168.77.51/255.255.255.0

The meaning of options rpcuser, rpcpassword and rpcallowip are all the same with the config file above.

RPC call example

You can make remote RPC calls after the settings above. Here is an example: Suppose the Simplechain node is running on an Ubuntu machine, whose ip is 192.168.77.188, default port=3889. Now we try to make jsonrpc call in a remote macbook:

zhongwenbins-MacBook-Pro:~ zhongwenbin$ curl --user test:test1234 --data-binary '{"jsonrpc": "1.0", "id":"curltest", "method": "getinfo", "params": [] }' -H 'content-type: text/plain;' http://192.168.77.188:3889/
{"result":{"version":140300,"protocolversion":70016,"walletversion":130000,"balance":0.00000000,"stake":0.00000000,"blocks":12197,"timeoffset":0,"connections":8,"proxy":"","difficulty":{"proof-of-work":1.52587890625e-05,"proof-of-stake":650787.7561123729},"testnet":false,"moneysupply":100028788,"keypoololdest":1505186997,"keypoolsize":100,"paytxfee":0.00000000,"relayfee":0.00400000,"errors":""},"error":null,"id":"curltest"}

The result is identical with local call.

Nginx settings（optional）

As the example above, you might find that it is quite complicated to make remote jsonrpc, since you must include rpcuser, rpcpassword, as well as port in the command. If you don't want users to provide these parameters, we recommand using Nginx. The benefits for Nginx is not only about simplification, but also a good way to hide rpcuser, rpcpassword and port number. And you can filter some RPCs for security. Here we suppose the reader have installed Nginx and have basic knowledges about how to use it.

example:

  • Simplechain node is running on 192.168.77.188,
  • api proxy ip 192.168.77.51, with Nginx installed

Instructions：

1.Set the Simplechain.conf of Simplechain node, remember to add the proxy ip to rpcallowip e.g.：

rpcuser=test
rpcpassword=test1234
rpcallowip=192.168.77.51/255.255.255.0

2.Setup Nginx ：

    server {
        listen       80;
        server_name  localhost;

        location / {
            proxy_pass http://192.168.77.188:3889;  #to port 3889
            proxy_set_header Authorization "Basic dGVzdDp0ZXN0MTIzNA==";  # its the base64 encode of test:test1234
        }
    }

3.Then you can make remote rpc call through proxy, without rpcuser or rpcpassword. e.g.：

zhongwenbins-MacBook-Pro:~ zhongwenbin$ curl  --data-binary '{"jsonrpc": "1.0", "id":"curltest", "method": "getinfo", "params": [] }' -H 'content-type: text/plain;' http://192.168.77.51/
{"result":{"version":140300,"protocolversion":70016,"walletversion":130000,"balance":0.00000000,"stake":0.00000000,"blocks":12250,"timeoffset":0,"connections":8,"proxy":"","difficulty":{"proof-of-work":1.52587890625e-05,"proof-of-stake":651324.7815933984},"testnet":false,"moneysupply":100029000,"keypoololdest":1505186997,"keypoolsize":100,"paytxfee":0.00000000,"relayfee":0.00400000,"errors":""},"error":null,"id":"curltest"}

This is only a simple example. You might have different settings or tools instead.
